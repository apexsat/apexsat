Introduction¶

This project is an embedded software system designed for an STM32-based satellite. The documentation provides structured guidelines to ensure code maintainability, readability, and compliance with industry standards.

Getting Started¶

If you are new to the project, visit getting_started to learn about setting up the development environment.

Coding Conventions¶

The Coding Conventions section details variable naming rules, function structures, and macro definitions to maintain consistency across the project.

Code Documentation¶

To properly document code using Doxygen and Sphinx, refer to Code Examples for STM32 Programming.

Project Structure¶

For an overview of the codebase, key modules, and project organization, refer to project_guidelines.

Development Workflow¶

  • Debugging and Testing: Learn about unit tests, logging, and debugging tools in debugging_testing.

  • Build System & CI/CD: Understand how to compile, build, and integrate the software in build_system.

Additional References¶

  • Code Examples: Common patterns and best practices for STM32 programming can be found in code_examples.

  • Glossary and Definitions: Technical terminology used throughout the project is explained in glossary.
